# Animation Guidelines

## Overview

Animations in 3HD2Kcinema should:

* improve user experience
* feel cinematic
* remain subtle
* avoid distraction

Animations should support immersion without overwhelming the interface.

---

# Animation Goals

Animations should:

* feel smooth
* improve interaction feedback
* support UI clarity
* remain performant

Avoid:

* excessive motion
* long delays
* distracting effects

---

# Animation Technology

## Primary Library

```txt id="q4u7kp"
Framer Motion
```

---

# Motion Philosophy

Use animation to:

* guide attention
* improve transitions
* enhance interactions

Do not animate purely for decoration.

---

# Hover Animations

Hover effects should:

* remain subtle
* feel responsive
* avoid large movement

Examples:

* card scaling
* glow effects
* button highlights

---

# Page Transitions

Page transitions should:

* remain fast
* feel smooth
* avoid excessive complexity

Recommended:

* fade transitions
* subtle slide transitions

---

# Modal Animations

Modals should:

* fade in smoothly
* scale subtly
* use backdrop blur

Avoid:

* aggressive bounce effects
* excessive motion blur

---

# Seat Interaction Animations

Seat interactions should:

* update instantly
* clearly indicate state changes
* feel responsive

Examples:

* hover glow
* selected pulse
* lock state transition

---

# Loading Animations

Use:

* skeleton loaders
* smooth shimmer effects
* lightweight loading states

Avoid:

* flashing loaders
* blocking animations

---

# Performance Rules

Animations should:

* remain GPU-friendly
* minimize layout shifts
* avoid unnecessary rerenders

Avoid:

* animating large layouts repeatedly
* expensive animation loops

---

# Duration Guidelines

Recommended durations:

```txt id="z5v2rw"
150ms
200ms
300ms
```

Avoid extremely slow animations.

---

# Transition Philosophy

Good transitions should:

* feel natural
* support interaction flow
* avoid interrupting users

---

# Responsive Animation Rules

Animations should:

* remain smooth on mobile
* avoid excessive CPU usage
* scale appropriately across devices

---

# Accessibility Considerations

Animations should:

* avoid triggering discomfort
* remain subtle
* support reduced motion preferences when possible

---

# Future Improvements

Possible future upgrades:

* advanced page transitions
* cinematic intro animations
* reduced motion settings
* shared animation utilities
